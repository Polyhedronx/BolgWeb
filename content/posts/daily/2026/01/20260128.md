## 2. tmux 终端复用练习

- 方便分屏，不需要开多个终端；保护现场，脱离会话后即使 SSH 断开也在后台跑
- 会话 Session -> 窗口 Windows -> 窗格 Pane
- 为了避免与普通快捷键冲突，所有快捷键都要先按 Ctrl + b，这里简称为 `Prefix`
- 查看所有快捷键：`Prefix ?`
- 不小心按到 `Prefix` 键可以按 `esc` 退出

```bash
# 新建并命名会话
tmux new -s <会话名称>
# 临时离开回到主屏，但 tmux 里的任务还在跑
Prefix + d
# 查看所有会话
tmux ls
# 回到上一个会话
tmux a
# 回到指定的会话，此处括号是一个排版整体，不需要打括号
tmux a -t <会话名称>
# 一层一层退出
exit
# 删除会话
tmux kill-session -t <会话名称>

# 左右分屏
Prefix + %
# 上下分屏
Prefix + "
# 切换焦点
Prefix + <方向键>
# 窗格全屏为窗口/窗口缩小为窗格
Prefix + z
# 关闭当前窗格/窗口
Prefix + x

# 新建窗口
Prefix + c
# 切换下一个/上一个窗口
Prefix + n/p
# 通过编号跳转
Prefix + <编号>
```

- 如果想用鼠标可以这样：

```bash
vim ~/.tmux.conf

# 在编辑模式（按完esc按i打开）输入
set -g mouse on

# 退出流程
esc
:wq

# 进入 tmux 按 Prefix + : 进入命令模式
source-file ~/.tmux.conf
```

## 3. Vim 常用技巧

### 切换模式

**`esc` 键回退到普通模式：**

- **`/关键词` ：实用的搜索功能，按回车搜索匹配项，按 `n` 跳转下一个匹配项。**
- `gg` ：快速定位到文件头。
- `G` ：快速定位到文件尾。
- `o` ：在当前行下方插入新行并进入编辑模式。
- `O` ：在当前行上方插入新行并进入编辑模式。
- **`A` ：跳到这一行末尾并进入编辑模式。**
- **`dd` ：剪切当前行，前面加数字可以剪切多行。**
- `x` ：删除光标下的单个字符。
- `u` ：撤回。
- `Ctrl + r` ：重做，即撤回撤回。
- `yy` / `p` ：复制当前行，粘贴到下方。

普通模式下输入 `:` 进入命令模式：

- `数字` ：直接跳到第几行。
- `set nu` / `set nonu` ：显示 / 关闭行号。
- **`wq` ：保存并退出。**
- `q!` ：不保存直接退出。
- `w !sudo tee %` ：改完发现没权限保存，直接输入这一串。[^1]
- `!bash %` ：写完 bash 脚本想直接测试一下。
- `r /path/to/template` ：直接读取其他文件内容合并。
- `set paste` / `set nopaste` ：粘贴前后输入，防止乱码。

**普通模式下输入 `i` 进入编辑模式。**

### vim 简单配置

```bash
vi ~/.vimrc

syntax on          " 开启语法高亮（写脚本必备）
set nu             " 显示行号
set ts=4           " Tab 键占 4 个空格
set expandtab      " 把 Tab 转为空格（防止不同系统缩进错乱）
set cursorline     " 高亮当前行（方便找光标）
```

### 一键拉取 aimx/vimrc 配置

```bash
git clone --depth=1 https://github.com/amix/vimrc.git ~/.vim_runtime
sh ~/.vim_runtime/install_basic_vimrc.sh
```

或者直接使用 scp 传输到服务器：

```bash
scp <本地的.vimrc路径> <用户名>@<服务器IP>:~/.vimrc
```

## 4. Bash 脚本管理和使用

1. 在服务器建立 `~/scripts` 文件夹，或者放在 `/usr/local/bin`[^2]。

2. 如果是自建的文件夹，记得加入 `~/.bashrc` 的 `PATH` 中。

   ```bash
   vi ~/.bashrc

   export PATH="$HOME/scripts:$PATH"

   # 保存退出后执行
   source ~/.bashrc
   ```

3. 所有的脚本不带 `.sh` 后缀，记得赋予 `+x` 执行权限。

   ```bash
   chmod +x <脚本名称>
   ```

4. 在本地电脑备份，用 Git 统一管理，新服务器可以直接拉取。

5. 脚本内部开头写几行注释，便于管理。

   ```bash
   #!/bin/bash
   # 描述：这个脚本是用来自动清理 7 天前 Nginx 日志的
   # 作者：我自己
   # 时间：2026-01-28
   ```

6. 脚本太多可以写一个入口脚本，方便使用。

   ```bash
   #!/bin/bash
   # 描述：一个入口脚本，用于帮助使用改配置、看日志等功能的脚本
   # 作者：我自己
   # 时间：2026-01-28
   case $1 in
       "conf")  vi /etc/nginx/nginx.conf ;;
       "log")   tail -f /var/log/nginx/error.log ;;
       "restart") systemctl restart nginx ;;
       *)       echo "用法: run {conf|log|restart}" ;;
   esac
   ```





[^1]: `！` 表示执行外部命令，`%` 表示当前文件。
[^2]: 前者一般是个人自用的服务器，后者是全系统多用户公用。项目单独的脚本可以放在项目根目录下的 `scripts` 文件夹。
