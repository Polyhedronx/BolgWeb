## 1. WSL 的配置

```.wslcofig
[wsl2]

# 允许 WSL 嵌套虚拟化，比如在 WSL 里再开 Docker 或虚拟机
nestedVirtualization=true

# WSL 将直接看到 Windows 的网卡，端口完全共享
networkingMode=mirrored

# 解决很多 VPN 环境下 DNS 失效的问题
dnsTunneling=true

# 让 WSL 自动释放不用的内存给 Windows，防止内存占用过高
autoMemoryReclaim=gradual

# 让 Windows 防火墙规则对 WSL 生效
firewall=true

# 自动把 Windows 的代理设置同步到 Linux 里
autoProxy=true
```

配置完记得 powershell 运行 `wsl --shutdown` 刷新后再进入 `wsl`。

## 2. brew 初见

### 环境配置与包管理难题

> apt 包管理器安装的软件通常为了稳定性版本较低，无法使用很多好用的新功能，而如果自己下载二进制文件又很难管理更新、卸载和配置。
>
> 于是引入 brew 这一开源的包管理工具，用于负责所有高效工具，且工具都是最新的稳定版本。
>
> 推荐：apt 管理系统级软件，比如 git ，crul，wget，build-essential，ssh，docker等。homebrew 管理个人生产力工具，比如 bat，fzf，btop，python，node等。

- 安装 brew：

```bash
# 安装 brew 依赖
sudo apt update
sudo apt install build-essential procps curl file git

# 清除 apt 的缓存
sudo apt autoremove
sudo apt clean

# 官方安装脚本
/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"

# 将 brew 添加到 PATH，这里以 polyhedron 用户为例
(echo; echo 'eval "$(/home/linuxbrew/.linuxbrew/bin/brew shellenv)"') >> /home/polyhedron/.bashrc
eval "$(/home/linuxbrew/.linuxbrew/bin/brew shellenv)"
```

- 用 brew 管理软件：

```bash
# 搜索新软件
brew search <softwarename>

# 安装软件
brew install <softwarename>

# 更新所有软件
brew upgrade

# 卸载软件
brew uninstall <softwarename>

# 查看已安装的软件,可以加 --version 参数查看版本号
brew list

# 查看手动指定安装的软件，不包括底层依赖
brew leaves

# 安全清理
brew cleanup

# 自动诊断，比如某某软件停止维护
brew doctor
```

### 用户管理

虽然 Homebrew 安装在 `/home/linuxbrew/.linuxbrew` 这个目录下，但它的设计初衷就是让普通用户去使用它。安装脚本已经把这个目录的使用权限赋予了当前目录的普通用户。

## 3. Vim 使用场景：全选并操作

```bash
# 进入命令模式
esc

# 跳转第一行
gg

# 进入可视化选择模式
Shift + v

# 跳转最后一行
Shift + g

# 1. 此时已经全选，接下来删除
d

# 当然这个操作也可以直接，% 就是全文件的意思
esc + :%d

# 2. 此时已经全选，接下来复制到 vim 剪切板
y
```

## 5 .Vim 和 Tmux 的一些配置

### 关于 vim

先下载 vim-plug，目的是加载柔和的色彩主题：

```bash
# Linux / Macos
curl -fLo ~/.vim/autoload/plug.vim --create-dirs \
    https://raw.githubusercontent.com/junegunn/vim-plug/master/plug.vim

# Windows
iwr -useb https://raw.githubusercontent.com/junegunn/vim-plug/master/plug.vim |`
    ni "$HOME/vimfiles/autoload/plug.vim" -Force
```

然后修改 `~/.vimrc`：

- 具体文件存放在我的 Git 仓库里。
- 不用脚本自动下载的话而是复制粘贴的话，记得先 `:set paste` 然后按 `i` 再粘贴。

进入 vim，`:PlugInstall`，等待下载完成直接 `:q`退出安装界面就行。

### 关于 Tmux

新建 `.config/tmux/tumx.conf`：

- 具体文件存放在我的 Git 仓库里。
- 粘贴的注意事项和 `.vimrc` 一样。
- 如果 tmux 版本较旧可能不会找这个路径，最好是创建一个软链接到家目录：`ln -s ~/.config/tmux/tmux.conf ~/.tmux.conf`。

检查颜色更改是否生效：

```bash
echo $TERM
# 若输出为 xterm-256color 就代表生效了
```
