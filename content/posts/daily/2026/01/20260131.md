## 1. 正则表达式 RegEx

> 正则表达式的测试网站：https://regexr.com

### 匹配模式

```shell
# 全局模式，匹配整个文件的内容
/<someRegEx>/g

# 多行匹配模式，匹配每一行，如果使用 ^ 或 $ 要开启这个才能匹配多行的开头和结尾，否则只会匹配全文本的开头和结尾
/<someRegEx>/m
```

### 基本语法

```shell
# 精确匹配 所有 ab
/abc/

# 匹配所有 ab 后面连接任意字符
/ab./

# 匹配所有 ab 后面连接特定字符
/ab[c]/			# 所有 abc
/ab[cd]/		# 所有 abc 和 abd
/ab[a-z]/		# 所有 ab 后面跟 a-z 的小写字母的字符串
/ac[a-zA-Z]/

# 方括号内部取反 ^
/ab[^a-z]/		# 所有 ab 后面不跟 a-z 的小写字母的字符串

# 将常用的匹配项简化
\d -> [0-9]，即 digital
\D -> [^0-9]
\w -> [a-zA-Z0-9_]，即 word
\W -> [^a-zA-Z0-9_]
\s -> 空白字符，如制表符、空格，即 space
\S -> 非空白字符

# 转义字符，例如匹配所有 .
/\./
```

### 位置和边界

```shell
# 外部的 ^ 表示匹配开头，例如匹配所有在开头的 a
/^a/

# 外部的 $ 表示匹配结尾，例如匹配所有在结尾的 a
/a$/

# 组合使用 ^ 和 $ 可以起到匹配只有需要匹配的字符串的内容的行，以 abc 为例
/^abc$/

# \b ，即 boundary 用于匹配单词开头或结尾，例如
/\bin/		# 只想匹配在单词开头的 in
/in\b/		# 只想匹配在单词结尾的 in
/\bin\b/	# 匹配独立的 in
```

### 量词

```shell
# + 匹配后续出现 1-n 次的字符
/ab+/		# 可以匹配到 ab、abb、abbb等

# * 匹配后续出现 0-n 次的字符
/ab*/		# 可以匹配到 a、ab、abb等

# ? 匹配后续出现 0-1 次的字符
/ab?/		# 可以匹配到 a、ab

# {number} 匹配后续出现 number 次的字符
/ab{3}/		# 可以匹配到 abbb

# {number1,number2} 匹配后续出现 number1 - number2 次的字符
/ab{1,3}/	# 可以匹配到 ab、abbb
/ab{1,}/	# 贪婪匹配，这里就类似 /ab+/，没有终止条件
/ab{1,}?/	# 非贪婪匹配，只满足最小匹配要求，这里只能匹配到 ab
```

### 分组和引用

```shell
# 使用 () 来分组
/(ab){2}/		# ab 此时作为一个整体，匹配到 abab

# 使用 [] 来表示字符集合
/[Aa]bandon/	# 可以同时匹配到 Abandon 和 abandon
/a[/_-.]/		# 可以同时匹配到 a/ 和 a_ 和 a- 和 a.

# 使用 | 来表示 “或者”
/abandon|Abandon/		# 可以匹配到 abandon 和 Abandon

# 可以发现 (A|a) = [Aa]

# $number 表示引用分组，这里举一个处理日期的例子
# 用于匹配类似：2026.01.31 2026-01-31 2026_01_31 2026/01/31 20260131 的日期
/(\d{4})[.-_/]?(\d{1,2})[.-_/]?(\d{1,2})/

$2-$1-$3		# 表示统一输出为 01-31-2026 这样用 - 连接的格式

# 如果不需要捕获某个特定的分组，可以在括号里使用 ?: 以下例子就没有捕获第三组
/(\d{4})[.-_/]?(\d{1,2})[.-_/]?(?:\d{1,2})/

# 在正则表达式内部解引用使用 \number，比如想匹配开头和结尾的字母一样的字符串
/\b([a-z])[a-z]*\1\b/
```

## 2. Gitconfig 管理

### 基础操作

```shell
# 编辑当前用户全局 .gitconfig，这是最常用的
git config --global --edit
# 相应地，本地 --local，系统 --system

# 列出所有 gitconfig 并标注来源文件路径
git config --list --show-origin
```

### 查看和更改远程仓库 URL

```shell
# 查看当前仓库远程地址
git config --local remote.origin.url

#修改为 SSH 地址
git remote set-url origin git@github.com:USERNAME/REPO.git
```

## 3. 配置 SSH-Agent 以实现免密链接

### 对于 Windows Powershell

```powershell
# 注意：只有新机器需要生成新的 SSH 密钥对
ssh-keygen -t ed25519 -C "your_email@example.com"

# 管理员运行 Open SSH Agent
Start-Service ssh-agent

# 设置为开机自启动
Set-Service -Name ssh-agent -StartupType 'Automatic'

# 添加密钥
ssh-add $HOME\.ssh\id_ed25519

# 检查已添加的密钥
ssh-add -l

# 对于 Git，需要强制其全局使用 OpenSSH 客户端
git config --global core.sshCommand C:/Windows/System32/OpenSSH/ssh.exe

# 验证连接是否成功
ssh -T git@github.com
```

## 4. 重定向和管道

> 在命令行执行一个程序时，系统会自动打开三个管道：
>
> 1. 标准输入 stdin：编号为 0。通常是程序从键盘读取数据。
> 2. 标准输出 stdout：编号为 1。程序运行成功的结果会送到这里。
> 3. 标准错误 stderr：编号为 2。程序运行出错的警告会送到这里。

### 重定向

如果不想让结果显示在屏幕上，而是想保存到文件里，就用输出重定向。

- `>` ：覆盖写入。它会把输出送到文件。如果文件不存在，它会创建；如果文件已存在，它会**清空**原来的内容，再写入新内容。

  ```bash
  #  屏幕没反应，但生成了一个 hello.txt，内容是 "Hello World"
  echo "Hello World" > hello.txt

  # 内容变成了 "Hi World"
  echo "Hi World" > hello.txt

  # 默认只重定向标准输出，如果要只重定向错误，使用 2>
  # 会重定向输出到 output.log，错误到 error.log
  command > output.log 2> error.log

  # 把错误和正常输出都塞进同一个文文件
  command > all.log 2>&1
  ```

- `>>` ：追加写入。它不会清空文件，而是在文件的**末尾**添加新内容。

  ```bash
  # test.txt 会有两行内容
  echo "Line 1" > test.txt
  echo "Line 2" >> test.txt
  ```

- `<` ：输入重定向。让程序不再从键盘读取，而是从一个**文件**里读取内容。

   ```bash
   # wc -l 命令是用来统计行数的。平时输入 wc -l 后要手动打字，最后按 Ctrl+D 结束。
   # 使用 < 进行输入重定向，程序会读取 hello.txt 的内容并数出有多少行。
   wc -l < hello.txt
   ```

### 管道

管道的作用是：将前一个命令的标准输出，变成后一个命令的标准输入，不需要产生中间文件

```bash
# ls 列出 /etc 所有的文件，交给 grep 去筛选包含 "conf" 的行，再显示出来
ls /etc | grep "conf"

# 读取文件 -> 排序 -> 去掉重复行
cat names.txt | sort | uniq

# 读取日志文件 -> 找到包含 404 的行 -> 按字母排序 -> 结果保存到 error.txt
cat access.log | grep "404" | sort > errors.txt
```

## 5. 通配符

- `*` ：最常用的通配符，代表所有。

  ```bash
  # 列出当前目录下所有以 .txt 结尾的文件
  ls *.txt

  # 列出当前目录下所有以 a 开头的文件
  ls a*

  # 删除当前目录下所有非隐藏文件
  rm *
  ```
- `?` ：匹配恰好一个字符，通常用于知道长度但不清楚具体名字的文件。

  ```bash
  # 可以匹配到 test1.txt testA.txt 等，但是无法匹配到 test123.txt
  ls test?.txt

  # 匹配名字只有两个字符的文件
  ls ??
  ```

- `[]` ：类似正则表达式，表示一个筛选器。

  ```bash
  # 匹配 a.txt b.txt c.txt
  ls [abc].txt

  # 匹配所有以小写字母命名的 .txt 文件
  ls [a-z].txt

  # 匹配所有不以数字命名的 .log 文件
  ls [!0-9].log
  ```

## 6. Shell 中的引用

```shell
# 单引号内的所有内容都会原样输出，Shell 不会对其中的任何符号做特殊处理
echo 'Hello * $USER'	# 输出 Hello * $USER

# 双引号屏蔽大多数特殊符号（比如 * 和 ?），但允许变量替换和命令替换
echo "Hello * $USER"	# 如果现在是 root 用户，屏幕上会输出：Hello * root
ls "My Project/"		# 查找中间带空格的目录下的文件

# 反斜杠一如既往地是转义字符
echo \$USER
```

## 7. 遇到僵尸进程怎么办

> 僵尸进程的原理：子进程在消失之前没给父进程发 wait() 指令。

```bash
# ps aux 列出所有进程，僵尸进程的状态码是 Z(Zombie)，找带有 <defunct> 标志的进程
ps aux | grep 'Z'

# 找到原本应该管理该僵尸进程的父进程
ps -o ppid,stat,comm -p <ZombiePID>

# 如果父进程不重要，可以直接杀掉，僵尸进程会被 PID 1 接管并清理
kill -9 <FatherPID>
```

## 8. 文件的阅读

- `head -n 5` ：只看前 5 行。
- `tail -n 5` ：只看最后 5 行。
- `tail -f` ：实时监控文件新增内容，常用于看日志。
- `less` ：交互式阅读。按 Space 翻页，/ 搜索，q 退出。
- `less +F` ：进入类似 `tail -f` 的实时追踪模式，按 Ctrl+C 回到普通浏览模式。

## 9. 权限管理

```bash
# ls -l 示例输出，-l 参数代表 long format
-rw-r--r-- 1 root root 4096 Jan 31 20:00 hello.txt
```

1. `-rw-r--r--` ：文件类型和权限
   - 第一位：`-` 是普通文件，`d` 是目录，`l` 是软链接
   - 后面九位：每三位一组，分别是拥有者、所属组、其他人的权限，`r` = 读，`w` = 写，`x` = 执行
2. `1` ：硬链接计数
3. `root` ：文件拥有者
4. `root` ：文件所属组
5. `4096` ：文件所占字节数
6. `Jan 31 20:00` ：最后修改时间
7. `hello.txt` ：文件名

```bash
# change owner
sudo chown polyhedron:someUsefulScripts.sh

# change mode
# 数字法：r = 4，w = 2，x = 1
chmod 755 someUsefulScripts.sh	# 拥有者全权，其他人只能读和执行

# 符号法：u = 拥有者，g = 所属组，o = 其他人
chmod u+x someUsefulScripts.sh	# 给拥有者加执行权
```
