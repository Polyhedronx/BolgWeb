## 1. `grep` 和 `grep -E`

- `grep` ：默认模式，特殊符号要加反斜杠转义才生效。
- `grep -E` ：正则模式，不需要对正则表达式所使用的特殊符号转义。

```bash
# -i 即 ignore case，忽略大小写
grep -i "error" app.log

# -v 即 invert match，反向过滤
tail -f app.log | grep -v "DEBUG"

# -o 即 only matching，只输出匹配到的部分
# 这一段是从文本中提取所有的 IP 地址
grep -E -o "([0-9]{1,3}\.){3}[0-9]{1,3}" access.log
```

## 2. Cargo 初见

```shell
# 建立新的项目文件夹，会自动生成 cargo.toml 配置文件、target 编译产物文件夹、src 源代码文件夹
cargo new <projectname>

# 编译并执行，若已编译过并且源代码未曾修改，则直接执行
cargo run

# 重要：检查代码能否通过编译，但不实际产生编译文件
cargo check

# 为发布时构建可执行文件，不包含调试代码和信息，可执行文件在 target/release 目录下
cargo build --release
```

## 3. Rust 初见

```rust
// 当使用 .method_name() 语法调用方法时引入换行符和空格将长的代码行拆开是明智的
std::io::stdin()
	// read_line 会返回一个类型为 Result 的值。Result 是一种枚举类型，通常也写作 enum
	// Result 的成员是 Ok 和 Err
    .read_line(&mut guess)
	// 如果 io::Result 实例的值是 Err，expect 会导致程序崩溃，并输出当做参数传递给 expect 的信息
    .expect("Failed to read line");
```

## 4. cargo.toml 的依赖库

```toml
[dependencies]
rand = "0.8.5"
# "0.8.5" 是 "^0.8.5" 的缩写，表示任何最低是 0.8.5 版本但不高于 0.9.0 的依赖库

# 在 build 时，不仅会引入 rand 库，还会引入 rand 库所需的依赖库，可以在自动生成的 cargo.lock 文件中看到
```

## 5. 总是忘记的 Git 撤回操作

```shell
# 丢弃工作区所有更改
git checkout .

# 丢弃工作区指定文件更改
git checkout <filepathname>

# 丢弃暂存区所有更改
git reset HEAD .

# 丢弃暂存区指定文件更改
git reset HEAD <filepathname>

# 回退一次 commit 提交到本地仓库的更改，不丢弃文件
git reset --soft HEAD^

# 回退到任意 commit 版本，不丢弃文件
git reset --soft commitid
```

## 6. 使用 uv 管理 Python 环境

```shell
# 创建名为 .venv 的虚拟环境
uv venv

# 激活环境（Windows）
.venv\Scripts\activate.ps1

# 安装最新版本
uv pip install requests

# 安装特定版本
uv pip install requests==2.31.0

# 从 requirements.txt 安装
uv pip install -r requirements.txt
```
