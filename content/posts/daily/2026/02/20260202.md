## 1. Rust 中的 Tuple 和 Array

- Tuple

```rust
// 显式指定 Tuple
let tup: (i32, f64, u8) = (10, 10.99, 1);

// 访问 Tuple：点标记法
println!("{}, {}, {}", tup.0, tup.1, tup.2);
```

- Array：存放在栈内存而非堆内存，元素固定。数组没有 Vector 灵活。

```rust
// 声明数组
let arr = [0, 1, 2, 3, 4];
let months = ["Jan","Feb", "Mar", "Apr", "May", "Jun", "Jul", "Aug", "Sep", "Oct", "Nev", "Dec"];

// 数组的类型 => [类型;长度]
let a:[i32;5] = [0, 1, 2, 3, 4];

// 数组元素相同 => [初始数据;长度]
// 下面这个等效于 [3, 3, 3, 3, 3]
let b = [3;5];

// 使用索引访问数组元素
// 访问索引超出数组范围，复杂情况编译会通过，运行时报错引起 panic，但不会允许继续访问相应地址的内存
let c = arr[0];
```

## 2. Shadowing

`mut` 与遮蔽的一个区别是，当再次使用 `let` 时，实际上创建了一个新变量，我们可以改变值的类型，并且复用这个名字。

```rust
fn main() {
    let x = 5;
    let x = x + 1;
    {
        let x = x * 2;
        println!("The value of x in the inner scope is: {x}");
    }
    println!("The value of x is: {x}");
}
```

```shell
$ cargo run
   Compiling variables v0.1.0 (file:///projects/variables)
    Finished `dev` profile [unoptimized + debuginfo] target(s) in 0.31s
     Running `target/debug/variables`
The value of x in the inner scope is: 12
The value of x is: 6
```

## 3. Rust 语句和表达式

`let y = 6` 语句并不返回值，所以没有可以绑定到 `x` 上的值。这与其他语言不同，例如 C 和 Ruby，它们的赋值语句会返回所赋的值。在这些语言中，可以这么写 `x = y = 6`，这样 `x` 和 `y` 的值都是 `6`；Rust 中不能这样写。

函数可以向调用它的代码返回值。我们并不对返回值命名，但要在箭头（`->`）后声明它的类型。在 Rust 中，函数的返回值等同于函数体最后一个表达式的值。使用 `return` 关键字和指定值，可从函数中提前返回；但大部分函数隐式的返回最后的表达式。这是一个有返回值的函数的例子：

```rust
fn five() -> i32 {
    // 如果加上分号把这个表达式变成语句，就会报错类型不匹配，因为语句只有一个返回值，就是()类型
    5
}
fn main() {
    let x = five();
    println!("The value of x is: {x}");
}
```

另一个帮助理解表达式的例子是：

```rust
fn main() {
    let condition = true;
    // 因为 if 是一个表达式，可以在 let 语句的右侧使用它，需要保证每个 arm 的类型相同
    let number = if condition { 5 } else { 6 };
    println!("The value of number is: {number}");
}
```

## 4. 循环标签

在很多编程语言中，使用 `break` 关键字只能跳出最内层循环，而在 rust 里可以使用循环标签来精准地跳出到对应的位置，比如下面的示例在最外层循环使用了循环标签 `'counting_up` 于是在 `break 'counting_up` 时就直接跳出最外层循环了。

```rust
fn main() {
    let mut count = 0;
    'counting_up: loop {
        println!("count = {count}");
        let mut remaining = 10;
        loop {
            println!("remaining = {remaining}");
            if remaining == 9 {
                break;
            }
            if count == 2 {
                break 'counting_up;
            }
            remaining -= 1;
        }
        count += 1;
    }
    println!("End count = {count}");
}
```

## 5. Rust 中的变量移动

```rust
// s1 被无效化了
let s1 = String::from("hello");
let s2 = s1;

// s 被重新赋值了
let mut s = String::from("hello");
s = String::from("ahoy");
println!("{s}, world!");

// 确实需要拷贝堆上的数据，可以用 .clone() 方法
let s1 = String::from("hello");
let s2 = s1.clone();
println!("s1 = {s1}, s2 = {s2}");

// 对于一些简单的数据类型，是可以只拷贝栈上的数据的，在 rust 中实现了 copy 的 trait
// 所有整数类型，比如 u32。布尔类型，bool，它的值是 true 和 false。所有浮点数类型，比如 f64。字符类型，char。
// 元组，当且仅当其包含的类型也都实现 Copy 的时候。比如，(i32, i32) 实现了 Copy，但 (i32, String) 就没有。
let x = 5;
let y = x;
println!("x = {x}, y = {y}");
```

其它语言中的浅拷贝：拷贝指针、长度和容量而不拷贝数据。Rust 中的移动：拷贝指针、长度和容量，并且使原变量无效。

其它语言中的深拷贝：不仅拷贝指针、长度和容量，还复制指针指向的堆上数据。当 `s2` 和 `s1` 离开作用域，它们都会尝试释放相同的内存。这是一个叫做 **二次释放** 的错误。而 Rust 永远也不会自动创建数据的深拷贝。

## 6. Rust 工具链的安装

我原本是使用 Visual Studio 的 C++ 库和 SDK，但一旦 Visual Studio 出了一点小差错往往会导致很严重的编译问题，而且由于 Visual Studio 的系统注册表错综复杂，即使使用卸载重下也不干净、往往浪费大量时间，于是忍无可忍转向 GNU 工具链。

我常用的编辑器是 VS Code。对于 VS Code 来说，它并不关心底层是用 MSVC 还是 GNU，它只负责调用 cargo 命令。所以只要把默认工具链切过去，VS Code 里的 rust-analyzer 插件和终端都会自动同步。以下是我安装 GNU 工具链的过程：

```shell
# 1. 设置镜像源为清华源
$Env:RUSTUP_DIST_SERVER = "https://mirrors.tuna.tsinghua.edu.cn/rustup"
$Env:RUSTUP_UPDATE_ROOT = "https://mirrors.tuna.tsinghua.edu.cn/rustup/rustup"

# 2. 执行安装
rustup toolchain install stable-x86_64-pc-windows-gnu

# 3. 设为默认工具链
rustup default stable-x86_64-pc-windows-gnu

# 4. 验证是否下载成功和是否切换到 GNU 版本
rustc --version
rustup show

# 在 VS Code 中按 Ctrl + Shift + P
# 输入并选择：Rust Analyzer: Restart server
```

对于 Cargo 工具下载包慢的问题，可以通过修改配置文件解决：

```shell
# 配置文件路径，如果没有就新建 C:\Users\Username\.cargo\config.toml
[source.crates-io]
replace-with = 'tuna'

# 清华大学的稀疏索引，如果失效请通过互联网查找
[source.tuna]
registry = "sparse+https://mirrors.tuna.tsinghua.edu.cn/rustup/crates.io-index/"
```
