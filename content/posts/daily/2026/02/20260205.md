## 1. Rust 中的 vector

### 访问方法：

```rust
let v = vec![1, 2, 3, 4, 5];

/* 对于第一个 [] 方法，当引用一个不存在的元素时 Rust 会造成 panic。
此方法适用于希望在尝试访问 vector 末尾之外的元素时让程序直接崩溃的场景。 */
let third: &i32 = &v[2];
println!("The third element is {third}");

// 当使用索引作为参数调用 get 方法时，会得到一个可以用于 match 的 Option<&T>
let third: Option<&i32> = v.get(2);
/* 当 get 方法被传递了一个数组外的索引时，它不会 panic 而是返回 None
比因为输入错误而使程序崩溃要友好。 */
match third {
    Some(third) => println!("The third element is {third}"),
    None => println!("There is no third element."),
}
```

### 引用规则：

```rust
// 不能在相同作用域中同时存在可变和不可变引用
let mut v = vec![1, 2, 3, 4, 5];
let first = &v[0];
v.push(6);
println!("The first element is: {first}");

/* 在 vector 的结尾增加新元素时，在没有足够空间将所有元素依次相邻存放的情况下，
可能会要求分配新内存并将老的元素拷贝到新的空间中。
这时，第一个元素的引用就指向了被释放的内存。借用规则阻止程序陷入这种状况。 */

let v = vec![100, 32, 57];
for i in &v {
    println!("{i}");
}

// 为了修改可变引用所指向的值，在使用 += 运算符之前必须使用解引用运算符（*）获取 i 中的值
let mut v = vec![100, 32, 57];
for i in &mut v {
    *i += 50;
}
```

## 2. rustlings 中的 vector 小例子

```rust
fn vec_map(input: &[i32]) -> Vec<i32> {
    input
        .iter()
        .map(|element| {
            // 这里不用手动解引用是因为对于常见类型如 i32 实现了自动解引用和 copy trait
            element * 2
        })
        .collect()
    /* input 的类型是 &[i32]（切片的引用）。
	.iter() 产生的 element 是 &i32（指向原数据的引用）。
	但是，map 闭包返回的结果 element * 2 是一个全新的 i32 值。
	最后的 .collect() 会把这些全新的值收集进一个全新的 Vec<i32> 中。
	*/
}
```

## 3. 在某个函数取得所有权之后仍想使用原变量

在需要取得所有权的函数传参时，原变量后使用 `.clone()` ，例子：

```rust
fn move_semantics2() {
    let vec0 = vec![22, 44, 66];

    let vec1 = fill_vec(vec0.clone());

    assert_eq!(vec0, [22, 44, 66]);
    assert_eq!(vec1, [22, 44, 66, 88]);
}
```
