## 1. Claude Code 配置文件中 Project 字段过长解决方法

```powershell
# 创建一个 claude code 的工作区目录，需要用到的时候就创建软链接
# 这样就不用到处授权了

# Powershell 创建软链接的语法，此处是将 D:\05_Projects\502_Practice\Rust\rustlings
# 链接到 D:\05_Projects\501_Works\ClaudeWork\projects
New-Item -ItemType SymbolicLink `
  -Path "D:\05_Projects\501_Works\ClaudeWork\projects" `
  -Target "D:\05_Projects\502_Practice\Rust\rustlings"

# 确认删除的是不是软链接，在配置好的情况下也可以直接 ll 查看
Get-Item "D:\05_Projects\501_Works\ClaudeWork\projects" | Format-List Mode,LinkType,Target
# 预期输出：LinkType 这一栏是 SymbolicLink
Mode     : l----
LinkType : SymbolicLink
Target   : "D:\05_Projects\502_Practice\Rust\rustlings"

# 删除软链接的方法
Remove-Item "D:\05_Projects\501_Works\ClaudeWork\projects"
```

## 3. String 类型

### push_str 方法

```rust
// push_str 方法采用字符串 slice，因为我们并不需要获取参数的所有权
// push_str 的返回类型是 ()，它只是就地修改字符串，不返回修改后的值
let mut s = String::from("foo");
s.push_str("bar");
```

错误的示例：

```rust
let s: &str = "hello";
// 编译错误
s.push_str("world");
```

- `&str` 是不可变的字符串切片
- `push_str` 只存在于 `String`
- `String` 是可增长、可变的堆分配类型

所以需要先有一个 `String` ：

```rust
let mut s = s.to_string();
s.push_str(" world");
```

### + 运算符和 add 函数

`+` 运算符使用了 `add` 函数，这个函数签名看起来像这样：

```rust
fn add(self, s: &str) -> String {
```

```rust
let s1 = String::from("Hello, ");
let s2 = String::from("world!");
// 注意 s1 被移动了，不能继续使用
let s3 = s1 + &s2; 
```

首先，`s2` 使用了 `&`，意味着我们使用第二个字符串的**引用**与第一个字符串相加。这是因为 `add` 函数的 `s` 参数：只能将 `&str` 和 `String` 相加，不能将两个 `String` 值相加。不过等一下 —— `&s2` 的类型是 `&String`, 而不是 `add` 第二个参数所指定的 `&str`。

之所以能够在 `add` 调用中使用 `&s2` 是因为 `&String` 可以被 **强转**（*coerced*）成 `&str`。当`add`函数被调用时，Rust 使用了一个被称为 **Deref 强制转换**（*deref coercion*）的技术，实际上会把 `&s2` 转换为 `&s2[..]`。因为 `add` 没有获取参数的所有权，所以在这个操作后 `s2` 仍然是有效的 `String`。

### format 宏

如果想要级联多个字符串，`+` 运算符的行为就显得笨重了。对于更为复杂的字符串链接，可以使用 `format!` 宏：

```rust
let s1 = String::from("tic");
let s2 = String::from("tac");
let s3 = String::from("toe");

// 宏 format! 生成的代码使用引用因此不会获取任何参数的所有权。
let s = format!("{s1}-{s2}-{s3}");
```
