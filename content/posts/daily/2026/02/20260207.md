## 1. 从 `.as_bytes()` 和 `.bytes()` 观察 Rust 中 API 的区别

判断一个字符串是否以字节序列 `0xEF 0xBB 0xBF`（UTF-8 BOM）开头：

- 需要 `b[0]`，`b[1]`，`b[2]`
- 需要一次性检查一段连续内存
- 用 `bytes()` 会迫使你写状态机或提前 collect

```rust
fn has_bom(s: &str) -> bool {
    let b = s.as_bytes();
    b.len() >= 3 && b[0] == 0xEF && b[1] == 0xBB && b[2] == 0xBF
}
```

统计字符串中 ASCII 控制字符（`< 0x20`）的数量：

- 不需要索引和切片
- 不需要保存中间结果
- 逻辑是流式判断每个字节

```rust
fn count_control(s: &str) -> usize {
    s.bytes().filter(|&b| b < 0x20).count()
}
```

## 2. `.replace()` 方法

- `input` 的类型是 `&str`，不可变、不可修改
- `replace` 是 `str` 的方法，**不会原地修改**
- `replace` 返回一个新的 `String`
- 返回值类型正好匹配 `-> String`

```rust
fn replace_me(input: &str) -> String {
    input.replace("cars", "balloons")
}

// 如果只想替换第一个
fn replace_me(input: &str) -> String {
    input.replacen("cars", "balloons", 1)
}
```

## 3. Rust 中管理不同的模块

- **内部使用**：`use path::Item;`
- **对外暴露 API**：`pub use path::Item;`
- **对外暴露 + 改名**：`pub use path::Item as Name;`

以下是 `rustlings` 的习题：

```rust
mod delicious_snacks {

    // 将子模块中的常量引入当前模块，并重命名
    pub use self::fruits::PEAR as fruit;
    pub use self::veggies::CUCUMBER as veggie;

    pub mod fruits {
        pub const PEAR: &str = "Pear";
        pub const APPLE: &str = "Apple";
    }

    pub mod veggies {
        pub const CUCUMBER: &str = "Cucumber";
        pub const CARROT: &str = "Carrot";
    }
}

fn main() {
    println!(
        "favorite snacks: {} and {}",
        delicious_snacks::fruit,
        delicious_snacks::veggie,
    );
}
```

## 4. `use` 关键字

`use` 后面允许的是一种树状结构，而不是并列的路径：

```rust
// 使用路径分组（最常见、也是推荐方式）
use std::time::{SystemTime, UNIX_EPOCH};
```

这里的结构是：

```shell
std::time
 ├── SystemTime
 └── UNIX_EPOCH
```

这样写的原因是：

- **语法更一致**：`use`、`pub use`、`use ... as ...`、`use ...::*` 都统一在同一种树结构下。
- **支持复杂重命名和嵌套**：比如下面这种写法，如果允许顶层逗号，语法会变得含糊且难解析。

  ```rust
  use std::{
      io::{self, Read, Write},
      fs::File,
  };
  ```
- **更利于工具和宏处理**：`use-tree` 是一个明确的 AST 结构，而不是语句列表。
