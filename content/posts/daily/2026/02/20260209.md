## 1. 从 `.entry()` API 看 Rust 的声明式语义

```rust
fn fruit_basket(basket: &mut HashMap<Fruit, u32>) {
    let fruit_kinds = [
        Fruit::Apple,
        Fruit::Banana,
        Fruit::Mango,
        Fruit::Lychee,
        Fruit::Pineapple,
    ];

    for fruit in fruit_kinds {
        basket.entry(fruit).or_insert(1);
    }
}
```

`.entry()` 和 `or_insert(1)` 的语义是：

> 给我这个 key 对应的值；如果没有，就创建一个。
>
> 如果 key 不存在，就插入 1；存在就什么都不做。

不需要关心之前是否存在；不会发生检查了不存在，但插入时已经存在的歧义。这就是 Rust 更推崇的 **声明式语义**。
