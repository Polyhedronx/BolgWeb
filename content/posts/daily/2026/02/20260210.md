## 1. rustlings 里的 quiz2 的小错误

- **把 `for` 当成有返回值的表达式**：`for` 永远返回 `()`，只能用于**副作用**（push、修改等）。必须自己创建 `Vec`，最后显式返回它。

- **花括号层级出错，函数被提前结束**：`output` 已经不在函数体里了。编译器提示 `expected one of ! or ::`，说明写了模块级代码。

- **把 `Vec<(A, B)>` 写成了 `Vec<A, B>`**： 以为 `Vec` 能装多个类型。`Vec<T>`：元素类型；`Vec<T, A>`：`A` 是 **allocator**，不是第二个元素类型，多个值必须用 **tuple**。比如 `Vec<(String, u32)>`

## 2. `{:?}` 的作用

含义是以 `debug` 模式打印变量，对于自定义类型来说，其类型定义上方必须加上 `#[derive(debug)]`。

和 `{}` 的区别在于，`{}` 是给用户看的格式化呈现，`{:?}` 用于开发者调试，多为自动生成。

## 3. `if let`

```rust
// match 表达式的情况
let mut count = 0;
match coin {
    Coin::Quarter(state) => println!("State quarter from {state:?}!"),
    _ => count += 1,
}

// if let 表达式的情况
let mut count = 0;
if let Coin::Quarter(state) = coin {
    println!("State quarter from {state:?}!");
} else {
    count += 1;
}
```

`if let 模式 = 表达式` 的内涵是模式在左、数据在右，是一个尝试解构的过程，同时也防止编译器以为在赋值变量。

## 4. `&` 和 `ref` 的差别

### `&` - 匹配的类型是引用

```rust
// x 的类型是 &i32
let x = &5;

match x {
    // 模式 &v 表示：期望被匹配的是一个引用，并把它解引用后绑定给 v
    &v => println!("{}", v),
}

// 概念上等价于
match &i32 {
    &i32 => i32
}
```

### `ref` - 决定绑定的是引用

```rust
struct Foo {
    x: i32,
}

let foo = Foo { x: 10 };

match foo {
    Foo { x } => { println!("{foo:?}") }
}

// 此时这里无法使用 foo，因为所有权在 match 表达式中已经被使用
println!("{foo:?}")

// 而如果使用 ref，则可以继续使用
// match foo {
//     Foo { ref x } => { println!("{foo:?}") }
//}
```
