## 2. 深入理解 `&str` 和 `&String`

| **特性**     | **String (通常以 &String 形式出现)** | **&str (字符串切片)**                                       |
| ------------ | ------------------------------------ | ----------------------------------------------------------- |
| **本质**     | 拥有所有权的堆分配字符串。           | 对字符串数据的**引用**（不持有所有权）。                    |
| **内存位置** | 数据存储在堆（Heap）上。             | 可以指向堆（来自 `String`），也可以指向静态内存（字面量）。 |
| **可变性**   | 可以动态增加或减少长度。             | 固定长度，不可变。                                          |
| **构成**     | 指针、长度、容量。                   | 指针、长度。                                                |

```rust
// 局限性大：只能接受 &String
fn direct_ref(s: &String) {
    println!("{}", s);
}

// 灵活性高：既能接受 &String，也能接受 &str
fn smart_ref(s: &str) {
    println!("{}", s);
}
```

 Rust 拥有 Deref 隐式转换。`&String` 会自动转换为 `&str`。如果用 `&str`，用户可以传入 `&String`、`"literal"` 或者 `s[0..5]`。如果用 `&String`，用户必须先创建一个 `String` 对象才能传参，这在处理字面量时会造成不必要的内存分配。

## 3. 再次理解引用的作用域

```rust
fn main() {
   let mut s = String::from("hello");

    let r1 = &s;
    let r2 = &s;
    println!("{} and {}", r1, r2);
    // 新编译器中，r1,r2作用域在这里结束

    let r3 = &mut s;
    println!("{}", r3);
} // 老编译器中，r1、r2、r3作用域在这里结束
  // 新编译器中，r3作用域在这里结束
```

在老版本的编译器中（Rust 1.31 前），将会报错，因为 `r1` 和 `r2` 的作用域在花括号 `}` 处结束，那么 `r3` 的借用就会触发 **无法同时借用可变和不可变** 的规则。

但是在新的编译器中，该代码将顺利通过，因为 **引用作用域的结束位置从花括号变成最后一次使用的位置**，因此 `r1` 借用和 `r2` 借用在 `println!` 后，就结束了，此时 `r3` 可以顺利借用到可变引用。

## 5. `enum` 的不引人注意的特性

无字段枚举（fieldless enum）在 Rust 中默认有整数判别值（discriminant），即使没有显式写出来。

```rust
enum Number {
    Zero,
    One,
    Two,
}
// 在 Rust 中，如果没有手动指定判别值：第一个变体默认是 0，后续依次递增
enum Number1 {
    Zero = 0,
    One,
    Two,
}
// C-like enum
enum Number2 {
    Zero = 0,
    One = 1,
    Two = 2,
}
fn main() {
    // as u8 在这里做的事情是：读取枚举的 discriminant（判别值）转换为整数类型
    // 只有无字段的枚举才能这么用
    assert_eq!(Number::One as u8, Number1::One as u8);
    assert_eq!(Number1::One as u8, Number2::One as u8);
} 
```
