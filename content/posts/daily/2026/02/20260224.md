## 1. Github 仓库拒绝 ssh 连接的解决方案

> 以前我为了 push 自己的代码仓库方便，将 git 的全局配置为了强制走 ssh 而不是 https，在配置 neovim 的时候出现了怎么也 clone 不了插件仓库的情况。这时我查找了包括 ssh 配置不一致、git 版本不一致、neovim 的 lua 配置出错的所有情况，最后依然解决不了，于是选择了走 https 连接，一下就治好了。这里记录一下解决方案。

```powershell
# 查找 git config 中关于 url 配置的项
git config --global --get-regexp url

# 如果输出类似 url.git@github.com:.insteadOf https://github.com/
git config --global --unset url.git@github.com:.insteadOf

# 配置自己的仓库走 ssh
git config --global url."git@github.com:yourname/".insteadOf "https://github.com/yourname/"
```
