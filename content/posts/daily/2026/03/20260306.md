## 1. PostgreSQL 实现 leetcode 1661.每台机器的进程平均运行时间

```postgresql
示例 1:

输入：
Activity table:
+------------+------------+---------------+-----------+
| machine_id | process_id | activity_type | timestamp |
+------------+------------+---------------+-----------+
| 0          | 0          | start         | 0.712     |
| 0          | 0          | end           | 1.520     |
| 0          | 1          | start         | 3.140     |
| 0          | 1          | end           | 4.120     |
| 1          | 0          | start         | 0.550     |
| 1          | 0          | end           | 1.550     |
| 1          | 1          | start         | 0.430     |
| 1          | 1          | end           | 1.420     |
| 2          | 0          | start         | 4.100     |
| 2          | 0          | end           | 4.512     |
| 2          | 1          | start         | 2.500     |
| 2          | 1          | end           | 5.000     |
+------------+------------+---------------+-----------+
输出：
+------------+-----------------+
| machine_id | processing_time |
+------------+-----------------+
| 0          | 0.894           |
| 1          | 0.995           |
| 2          | 1.456           |
+------------+-----------------+
解释：
一共有3台机器,每台机器运行着两个进程.
机器 0 的平均耗时: ((1.520 - 0.712) + (4.120 - 3.140)) / 2 = 0.894
机器 1 的平均耗时: ((1.550 - 0.550) + (1.420 - 0.430)) / 2 = 0.995
机器 2 的平均耗时: ((4.512 - 4.100) + (5.000 - 2.500)) / 2 = 1.456
```

以上是题目描述。

> 我看到这一题的思路是先通过自联表把每个进程的 start 和 end 放在同行不同列中，方便直接用 AVG() 函数求得 timestamp 的平均值。

```postgresql
select
    a.machine_id,
    ROUND(AVG(b.timestamp - a.timestamp), 3) as processing_time
from Activity a
join Activity b
    on a.machine_id = b.machine_id
    and a.process_id = b.process_id
where
    a.activity_type = 'start'
    and b.activity_type = 'end'
group by
    a.machine_id
order by
    machine_id asc
```

## 2. leetcode 1280. 学生们参加各种考试的次数

解题的路线是：

1. 先看需求输出的表格格式，发现每个人都对应了所有的考试科目，即使有的人没考某些科目。很自然地想到使用 `CROSS JOIN` 来实现这种笛卡尔积。
2. 想办法将参加考试的情况往上面凑出来的好形式凑，想着如何不丢失考试情况的信息，就将上面凑出来的表 `LEFT JOIN` 到考试情况表上，这里使用了子查询。
3. 这时优先 `GROUP BY` + `ORDER BY` 整理完 `SELECT *` 看看形式，发现已经很接近需求，只需要 `COUNT()` 一下就行。

```postgresql
-- 最终的题解
select
    t.student_id,
    t.student_name,
    t.subject_name,
    count(ex.subject_name) as attended_exams
from
    (select * from Students cross join Subjects ) t
    left join Examinations ex
        on ex.student_id = t.student_id
        and t.subject_name = ex.subject_name
group by
    t.student_id,
    t.student_name,
    t.subject_name
order by
    t.student_id,
    t.subject_name
```
