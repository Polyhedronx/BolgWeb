## 1. leetcode 1633. 各赛事用户的注册率

```postgresql
select
    contest_id,
    coalesce(
        round(
            -- cast(... as decimal) 的意思是将 ... 转为 decimal 格式，便于浮点数计算，是标准 SQL 写法。PostgreSQL 可以直接在后面加 ::decimal 或 ::numeric
            -- (select count(*) from users) 是一个子查询，这样就不用联表了
            cast(100 * count(*) as decimal) / (select count(*) from users)
        ,2 )
    ,0 ) as percentage
from
    Register
group by
    contest_id
order by
    percentage desc,
    contest_id asc
```

## 2. SQL 对日期的处理

刷题的时候发现不会将 date 或者 timestamp 格式的数据转成对应精度的日期，使用 date_trunc() 函数会在后面拖一个类似 00:00:00 的尾巴，于是查了查。

```postgresql
SELECT TO_CHAR(CURRENT_TIMESTAMP, 'YYYY-MM-DD HH24:MI:SS');

-- 其中 ‘YYYY-MM’ 这一段可以自由组合
```

>  注意：PostgreSQL 中 GROUP BY 不能使用别名，能使用的是 MySQL。

## 3. CASE 语句的一些特点

```postgresql
COUNT(CASE WHEN state = 'approved' THEN 1 END)
SUM(CASE WHEN state = 'approved' THEN 1 END)   -- 不满足时返回 NULL，SUM 忽略 NULL
```

- `CASE` 在满足条件时返回 `1`，不满足时返回 `NULL`。
- `COUNT` 和 `SUM` 只统计 非 `NULL` 值的出现次数，因此等价于统计 `state = 'approved'` 的行数。

## 4. 查询最早日期和子查询的一些心得

使用 `MIN()` 来查找最早日期。

子查询可以单独提出来成为一张表，增加可读性，同时也便于测试和维护，使用类似如下语法：

```postgresql
WITH Tablename AS (
	SELECT ...
    FROM ...
    GROUP BY ...
)
```

注意尽量使用 **非相关** 子查询，不依赖外部查询列，对性能影响小。要使用相关子查询的情况可以改写为 `JOIN` 或窗口函数。以下给出一些示例：

```postgresql
-- 非相关子查询
SELECT employee_id, salary
FROM employees
WHERE salary > (SELECT AVG(salary) FROM employees);

-- 相关子查询
SELECT e1.employee_id, e1.salary, e1.department_id
FROM employees e1
WHERE salary > (SELECT AVG(salary)
                FROM employees e2
                WHERE e2.department_id = e1.department_id);
```

对于上面那个相关子查询有以下改写方法：

```postgresql
-- 使用 JOIN 加预分组计算的方法，先按部门计算出平均工资，然后与原表进行连接，再过滤。
WITH dept_avg AS (
    SELECT department_id, AVG(salary) AS avg_salary
    FROM employees
    GROUP BY department_id
)
SELECT e.employee_id, e.salary, e.department_id
FROM employees e
JOIN dept_avg d ON e.department_id = d.department_id
WHERE e.salary > d.avg_salary;

-- 用窗口函数 AVG() OVER(PARTITION BY ...) 直接为每一行附加部门平均工资，然后在外层过滤。
WITH salary_with_avg AS (
    SELECT 
        employee_id,
        salary,
        department_id,
        AVG(salary) OVER (PARTITION BY department_id) AS avg_salary
    FROM employees
)
SELECT employee_id, salary, department_id
FROM salary_with_avg
WHERE salary > avg_salary;
```
