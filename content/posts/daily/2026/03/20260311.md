## 1. `ROW_NUMBER()` 用法心得

作为一个窗口函数，用法和其他窗口函数一样：

```postgresql
ROW_NUMBER() OVER (
    PARTITION BY 分组字段
    ORDER BY 排序字段 DESC/ASC
) AS 别名
```

它的作用是给每一行一个连续且不重复的序号，通常用于：
- 取每组前 N 条
- 取最新 / 最旧 / 最大 / 最小的 N 条
- 去重、分页

相比之下，`RANK()` 会重复、会跳号，`DENSE_RANK()` 会重复、不跳号。下面是我使用该聚合函数完成 leetcode 1164. 指定日期的产品价格 的题解。

```postgresql
WITH Rank AS (
    SELECT
        product_id,
        new_price,
        ROW_NUMBER() OVER (PARTITION BY product_id ORDER BY change_date DESC) AS rk
    FROM Products
    WHERE change_date < '2019-08-17'
)

SELECT
    product_id,
    new_price AS price
FROM Rank
WHERE rk = 1

UNION

SELECT
    product_id,
    10 AS price
FROM Products
GROUP BY product_id
HAVING MIN(change_date) > '2019-08-16'
```

## 2. Bash 中统计词频

```bash
cat words.txt | tr -s ' ' '\n' | sort | uniq -c | sort -nr | awk '{ print $2, $1 }'
```

- 其中 `tr` 命令在 `20260304.md` 中的关于 Windows 在 SSH 终端环境查找环境变量时忘记使用 `-split` 参数时尝试过使用类似 `tr ';' '\n'` 的方法来为每个单独的环境变量换行独立显示，本质就是一个字符替换（当然也可以用于删除），加上 `-s` 参数是为了缩减连续的字符（这里是空格）为指定的单个字符（这里是换行符）。

- `uniq` 前需要先 `sort` 所以这里的 `sort` 不是多余的。加上 `-c` 参数可以将统计的词频数打印在前面，输出类似：

  ```shell
  1 day
  3 is
  2 sunny
  4 the
  ```

- 接下来就是需要按照词频数倒序排列了，此处刚好每行行首都是数字，直接 `sort -r` 就能方便地倒序排列。

> 但是注意：当单词的词频数超过 10 时，就不能单纯依靠 sort -r 的默认的 ASCII 码来排序，而是要使用 -nr 参数，表示按数字倒序排序。

- 最后是格式化打印，最常用的就是 `awk` ，注意给数字和单词换个位置就行。
