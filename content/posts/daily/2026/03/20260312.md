## 1. 从转置脚本中学习 AWK 中的内置变量

以下是 leetcode 194. 转置文件的个人题解，在解题的过程中，发现对 AWK 的内置变量的运用完全不熟悉，于是边查阅文档边在 WSL 里测试，最终完成了此题解：

```bash
awk '
{
    # 存储当前行的每个字段
    for (i = 1; i <= NF; i++) {
    	# 用 NR 作为行号，i 作为列号
        data[NR, i] = $i
    }
    # 更新最大列数
    if (NF > max_col) max_col = NF
}
END {
    # 按列输出：遍历每一列
    for (col = 1; col <= max_col; col++) {
        line = ""
        for (row = 1; row <= NR; row++) {
            val = data[row, col]
            # 拼接当前列的所有行值，用 OFS 分隔
            if (row == 1)
                line = val
            else
                line = line OFS val
        }
        # 输出时自动添加 ORS
        print line
    }
}' file.txt
```

### 内置变量的信息

| 变量         | 全称                        | 含义                                                         | 默认值                   | 在转置代码中的作用                                           |
| :----------- | :-------------------------- | :----------------------------------------------------------- | :----------------------- | :----------------------------------------------------------- |
| **NF**       | **Number of Fields**        | 当前记录（行）的字段个数（即列数）                           | 随输入行变化             | 用于获取当前行有多少列，从而循环遍历每个字段，并更新最大列数 `max_col`。 |
| **NR**       | **Number of Records**       | 当前已处理的记录数（即行号，从1开始）                        | 随输入行递增             | 用作数组的行索引，存储每个字段到二维数组 `data[NR,i]`；在 END 块中表示总行数，用于遍历所有行。 |
| **OFS**      | **Output Field Separator**  | 输出字段分隔符，当使用 `print` 打印多个参数时，参数之间会自动插入该字符串 | 空格（" "）              | 在拼接转置后的一行数据时，使用 `OFS` 连接各列的值，保证输出格式统一。 |
| **FS**       | **Field Separator**         | 输入字段分隔符，用于将一行分割成字段                         | 空白字符（空格或制表符） | 可通过 `-F` 选项或 `BEGIN` 块设置，决定如何拆分输入行。      |
| **RS**       | **Record Separator**        | 输入记录分隔符，用于分割不同的记录（行）                     | 换行符（"\n"）           | 控制 AWK 如何读取下一行，一般无需修改。                      |
| **ORS**      | **Output Record Separator** | 输出记录分隔符，每个 `print` 语句结束后自动输出该字符串      | 换行符（"\n"）           | 在 END 块的 `print line` 中自动添加换行，确保每列输出一行。  |
| **FILENAME** | **File Name**               | 当前正在处理的输入文件名                                     | 无                       | 可用于多文件处理时区分文件来源。                             |

### 各变量的作用详解

1. **NF**
   - 在每一行处理中，`NF` 给出当前行的字段数。
   - 循环 `for (i = 1; i <= NF; i++)` 确保遍历该行的所有字段。
   - 条件 `if (NF > max_col)` 用于记录所有行中最大的字段数，以便最后按此列数输出。
2. **NR**
   - 在处理第一行时，`NR = 1`；第二行时 `NR = 2`，以此类推。
   - 使用 `data[NR, i] = $i` 将每个字段存储在以行号为第一维、列号为第二维的数组中，这样后续可以按行号、列号访问。
   - 在 END 块中，`NR` 的值就是总行数，所以内层循环 `for (row = 1; row <= NR; row++)` 能够遍历所有行。
3. **OFS**
   - 在拼接 `line` 时，使用 `line OFS val`，这样每两个值之间会自动插入 `OFS` 的当前值（默认为空格）。
   - 如果希望输出制表符分隔，可以在命令行通过 `-v OFS='\t'` 设置，无需修改代码。
4. **ORS**
   - 虽然代码中没有显式使用 `ORS`，但 `print line` 语句在执行完毕后会自动输出 `ORS`（默认为换行），因此转置后的每一列会单独占一行。
5. **FS**（未在代码中直接出现，但可通过命令行指定）
   - 例如 `-F','` 会将输入字段分隔符改为逗号，从而正确解析 CSV 文件。

---

### 为什么需要这些变量？

- **NF** 提供了每行的列数，使得程序能够适应不同列数的行，而不必预先知道结构。
- **NR** 提供了行号，可以用来构建二维数组，并知道总共处理了多少行，便于后续遍历。
- **OFS** 和 **ORS** 让输出格式可控，可以灵活调整分隔符而不改动核心逻辑。

这些内置变量是 AWK 强大文本处理能力的基础，掌握它们能写出更通用、更简洁的脚本。

---

### 扩展：字段引用 `$i`

在代码中还有 `$i`，其中 `i` 是变量，`$i` 表示当前行的第 `i` 个字段。例如 `$1` 是第一个字段，`$NF` 是最后一个字段。当 `i` 变化时，`$i` 动态引用不同的字段。这是 AWK 的一个重要特性，允许通过变量间接访问字段。
