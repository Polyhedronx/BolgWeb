## 1. `SUBSTRING()` 的用法和优化

在考虑如何使用 SQL 语句将查询结果格式化为字符串特定位置大写或小写的时候，查到了这个函数，作用是截取特定位置的子字符串，时间复杂度是 O(n)：

```postgresql
substring(string [from start_position] [for length])
-- 或
substring(string, start_position, length)
```

- `string`：源字符串。
- `start_position`：起始位置（整数，从 1 计数）。如果为负数，则从末尾开始计数。
- `length`：要提取的字符数（可选，省略则提取到末尾）。

以下是示例：

```postgresql
SELECT substring('PostgreSQL' from 5 for 2);      -- 结果：'gr'
SELECT substring('PostgreSQL', 5, 2);             -- 结果：'gr'
SELECT substring('PostgreSQL' from 5);            -- 结果：'greSQL'
SELECT substring('PostgreSQL' from -4 for 3);     -- 从末尾第4个开始取3个字符 → 'SQL'
```

## 2. 连接字符串使用 `CONCAT()` 或 `||`

以下是 `CONCAT` 基本语法和示例：

- 参数可以是任意数量，自动将参数转换为字符串类型（如果参数不是字符串，会尝试隐式转换）。
- 如果任一参数为 **NULL**，`CONCAT` 会将其视为空字符串，而**不会**导致整个结果为 NULL。

```postgresql
CONCAT(str1, str2, ...)

SELECT CONCAT('Hello', ' ', 'World');   -- 结果：'Hello World'
SELECT CONCAT('Postgre', NULL, 'SQL');  -- 结果：'PostgreSQL'（NULL 被忽略）
SELECT CONCAT('Value: ', 123);           -- 结果：'Value: 123'（数字自动转换）
```

不过如果是在 SQL 中，其实一般情况下更适合的方案是使用 `||`，因为处理 NULL 的方式不同——一旦有任意字符串是 NULL，则返回的表达式结果也是 NULL。

```postgresql
SELECT 'Hello' || ' ' || 'World';       -- 结果：'Hello World'
SELECT 'Postgre' || NULL || 'SQL';       -- 结果：NULL
```

还有一种带分隔符的版本，也就是 `CONCAT_WS`：

```postgresql
SELECT CONCAT_WS(', ', 'apple', 'banana', 'cherry');  -- 结果：'apple, banana, cherry'
SELECT CONCAT_WS('-', '2023', NULL, '01');            -- 结果：'2023-01'（NULL 被跳过）
```

## 3. PostgreSQL 中使用正则的方法

下面是一些常用的正则表达式函数及其用法：

- `~`: 包含正则表达式模式的匹配。
- `~*`: 不区分大小写的包含正则表达式模式的匹配。
- `!~`: 不匹配正则表达式模式。
- `!~*`: 不区分大小写的不匹配正则表达式模式。

```postgresql
-- 查找以"abc"开头的字符串
SELECT 'abcdef' ~ '^abc'; -- 返回true

-- 查找包含"123"的字符串
SELECT 'abc123def' ~ '123'; -- 返回true

-- 查找不包含"abc"的字符串
SELECT 'abcdef' !~ 'abc'; -- 返回false

-- 忽略大小写查找包含"xyz"的字符串
SELECT 'abcXYZdef' ~* 'xyz'; -- 返回true

```

---

### 匹配重复的字符

正则表达式中的量词可以指定任意字符重复出现的次数。下面是一些示例：

```sql
-- 查找重复出现的数字
SELECT '1234' ~ '(\d)\1'; -- 返回true

-- 查找长度为3的连续字母
SELECT 'abc123' ~ '([a-z])\1\1'; -- 返回true
```

---

### 提取匹配的子字符串

使用圆括号`()`可以捕获子字符串，然后我们可以通过反向引用`\`来引用它们。下面是一个示例：

```sql
-- 提取邮政编码
SELECT substring('邮政编码：123456' from '邮政编码：(\d+)'); -- 返回123456
```

---

### 替换字符串

使用正则表达式替换函数`regexp_replace`，可以非常方便地替换匹配的字符串。下面是一个示例：

```sql
-- 替换连续的数字为"X"
SELECT regexp_replace('12-34-56', '\d+', 'X'); -- 返回"X-X-X"
```

## 4. 再度温习一下 PosqtgreSQL 的执行顺序

```mermaid
graph LR
    A[FROM/JOIN] --> B[WHERE]
    B --> C[GROUP BY]
    C --> D[HAVING]
    D --> E[SELECT]
    E --> F[DISTINCT]
    F --> G[ORDER BY]
    G --> H[LIMIT/OFFSET]
```

- **FROM/JOIN**：确定数据源，包括表连接和派生表。
- **WHERE**：过滤行（不能使用聚合函数或别名）。
- **GROUP BY**：将行分组。
- **HAVING**：过滤分组（可使用聚合函数）。
- **窗口函数**：在 `SELECT` 阶段计算，但逻辑上在 `ORDER BY` 之前。
- **SELECT**：选择列、计算表达式、分配别名。
- **DISTINCT**：去除重复行。
- **集合操作**（UNION 等）：合并多个查询结果。
- **ORDER BY**：排序（可使用别名或列序号）。
- **LIMIT/OFFSET**：限制返回行数。
