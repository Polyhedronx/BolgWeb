## 1. 与 MySQL 中 `GROUP_CONCAT` 类似的 PostgreSQL 中的 `STRING_AGG`

假设有一张 Students 表：

| id   | name | class | score |
| ---- | ---- | ----- | ----- |
| 1    | 张三 | 一班  | 90    |
| 2    | 李四 | 一班  | 85    |
| 3    | 王五 | 二班  | 95    |
| 4    | 赵六 | 二班  | 88    |

---

### 分组拼接

按班级分组，拼接学生姓名，对标 `GROUP_CONCAT` 基础用法：

```postgresql
SELECT 
  class,
  string_agg(name, ',') AS student_names  -- 用逗号分隔姓名
  -- 如果需要去重，直接在列名之前加 DISTINCT 就行
FROM student
GROUP BY class;
```

此时输出结果为：

| class | student_names |
| ----- | ------------- |
| 一班  | 张三，李四    |
| 二班  | 王五，赵六    |

---

### 函数内排序

拼接时按分数降序排序，PostgreSQL 的扩展优势：

```postgresql
SELECT 
  class,
  string_agg(name, ',' ORDER BY score DESC) AS student_names
FROM student
GROUP BY class;
```

此时输出结果为：

| class | student_names |
| ----- | ------------- |
| 一班  | 张三，李四    |
| 二班  | 王五，赵六    |

---

### 格式化拼接多字段

```postgresql
SELECT 
  class,
  string_agg(concat(name, '-', score), ',') AS name_score
FROM student
GROUP BY class;
```

此时输出结果为：

| class | name_score           |
| ----- | -------------------- |
| 一班  | 张三 - 90, 李四 - 85 |
| 二班  | 王五 - 95, 赵六 - 88 |
