## 1. `.iter()` 和 `.enumerate()` 为何如此常见？

#### 1. `iter()`：安全的只读遍历

`iter()` 是 Rust 集合（`Vec`、`&[u8]`、`String` 转的字节数组等）的核心迭代器方法，作用是：

- 创建一个**不可变迭代器**（`Iterator<Item = &T>`），遍历集合时**仅借用元素**，不获取所有权，遍历后原集合仍可正常使用；
- 避免不必要的拷贝 / 移动：比如遍历 `&[u8]` 时，`iter()` 直接返回元素的引用（`&u8`），而非复制字节值，效率极高；
- 符合 Rust 所有权规则：Rust 最容易踩的坑是遍历后集合被消费无法使用，而 `iter()` 因为只是借用，完全规避了这个问题。

#### 2. `enumerate()`：迭代器的索引包装

`enumerate()` 是所有迭代器都实现的方法（因为它是 `Iterator` trait 的方法），作用是：

- 把原迭代器的每个元素，包装成一个元组 `(索引, 元素引用)`；
- 索引从 `0` 开始，自动递增，无需手动维护；
- 不改变原迭代器的遍历逻辑，只是附加了位置信息。

## 2. [438. 找到字符串中所有字母异位词](https://leetcode.cn/problems/find-all-anagrams-in-a-string/)

我的思路历程如下：

1. 用写过的判断异位词的方法，整合出一个滑动窗口逐级寻找。缺点是每次滑动都要计算质数积和模数，滑动中又要用到费马小定理来取模，时间复杂度高且心智负担重。
2. 转换到用数组来进行词频统计的思路，此时思考如何比对更快，而不是每次都比对整个词频数组。最终想的是初始化窗口的时候加入一个变量 `target_match` 来表示需要比对的字符数，在滑动窗口时维护一个 `current_match` 变量，当他们相等的时候就直接将索引 push 到 vec 中。**但是**，这样做有点反直觉，因为变量没有体现匹配逻辑，所有匹配逻辑都是在各种 if 语句里，而且每次都要冗余比较当前索引下的词频是否大于 0。代码如下：

```rust
impl Solution {
    pub fn find_anagrams(s: String, p: String) -> Vec<i32> {
        let (s, p) = (s.as_bytes(), p.as_bytes());
        let (s_len, p_len) = (s.len(), p.len());
        let mut ans = Vec::new();

        if s_len < p_len {
            return ans;
        }

        // 统计词频
        let mut win_freq = [0; 26];
        let mut p_freq = [0; 26];

        // 计算需要比对的字符数、同时初始化 p 的词频
        let target_match = {
            let mut cnt = 0;
            for &c in p {
                let idx = (c as usize - 'a' as usize);
                p_freq[idx] += 1;
                if p_freq[idx] == 1 {
                    cnt += 1;
                }
            }
            cnt
        };
        
        let mut cur_match = 0;

        // 初始化第一个窗口
        for i in 0..p_len {
            let idx = (s[i] as usize - 'a' as usize);
            if p_freq[idx] > 0 && win_freq[idx] == p_freq[idx] {
                cur_match -= 1;
            }
            win_freq[idx] += 1;
            if p_freq[idx] > 0 && win_freq[idx] == p_freq[idx] {
                cur_match += 1;
            }
        }
        
        if cur_match == target_match {
            ans.push(0);
        }

        // 滑动窗口
        for i in p_len..s_len {
            // 移除左边的字符
            let left_char = (s[i - p_len] as usize - 'a' as usize);
            // 移除前有匹配记录则去掉一次
            if p_freq[left_char] > 0 && win_freq[left_char] == p_freq[left_char] {
                cur_match -= 1;
            }
            // 减一次窗口中左边界的字符频率
            win_freq[left_char] -= 1;
            if p_freq[left_char] > 0 && win_freq[left_char] == p_freq[left_char] {
                cur_match += 1;
            }

            // 移入右边的字符
            let right_char = (s[i] as usize - 'a' as usize);
            if p_freq[right_char] > 0 && win_freq[right_char] == p_freq[right_char] {
                cur_match -= 1;
            }
            // 这里是加一次
            win_freq[right_char] += 1;
            if p_freq[right_char] > 0 && win_freq[right_char] == p_freq[right_char] {
                cur_match += 1;
            }

            // 判断是否匹配计数相等，相等则返回起始索引
            if cur_match == target_match {
                ans.push((left_idx + 1) as i32);
            }
        }

        ans
    }
}
```

3. 在 AC 之后我看了看 leetcode 官方的题解，整体滑动窗口的思路是一致的，但是官方值得学习的地方是：用**差异数量**来作为是否 push 的关键变量，只需要令 s 中出现的字符让词频数组改变的方向与 p 中出现的字符让词频数组改变的方向**相反**就行了，最后统计词频数组中**为 0 的**元素数量，如果也是 0 则完全匹配。这个方法实在很优雅，官方没有 Rust 版的题解，我的代码如下：

```rust
impl Solution {
    pub fn find_anagrams(s: String, p: String) -> Vec<i32> {
        let (s, p) = (s.as_bytes(), p.as_bytes());
        let (s_len, p_len) = (s.len(), p.len());

        if s_len < p_len {
            return Vec::new();
        }

        let mut ans = Vec::new();
        let mut count = [0i32; 26];

        // 初始化窗口
        for i in 0..p_len {
            // s 中出现的字符加 1
            let s_char = (s[i] - b'a') as usize;
            count[s_char] += 1;
            // p 中出现的字符减 1
            let p_char = (p[i] - b'a') as usize;
            count[p_char] -= 1;
        }

        let mut diff = count.iter().filter(|&&x| x != 0).count();

        if diff == 0 {
            ans.push(0);
        }

        // 滑动窗口，左侧移除字符，右侧增加字符，处理 count 和 diff 状态
        for i in 0..(s_len - p_len) {
            let left_char = (s[i] - b'a') as usize;
            if count[left_char] == 1 {
                diff -= 1;
            } else if count[left_char] == 0 {
                diff += 1;
            }
            count[left_char] -= 1;

            let right_char = (s[i + p_len] - b'a') as usize;
            if count[right_char] == -1 {
                diff -= 1;
            } else if count[right_char] == 0 {
                diff += 1;
            }
            count[right_char] += 1;

            if diff == 0 {
                ans.push((i + 1) as i32);
            }
        }

        ans
    }
}
```
