## 1. 如何从轮转数组自然而然地想到反转三次数组？

问题：给定数组 `nums = [1,2,3,4,5,6,7]`，向右轮转 `k=3` 位，结果是 `[5,6,7,1,2,3,4]`。

- 第一个想法是通过取模算出新位置，然后创建新数组，对应好下标就行了，但是这样空间复杂度是 $O(n)$，于是思考是不是有什么更优的方法。
- 如何让 `5,6,7` 到数组的前面而 `1,2,3,4` 到数组的后面呢？拆数组？这样空间复杂度也不行。扩容数组？一样的。
- 想着用一个变量寄存取出来的元素，然后将应该变动到那个位置的元素塞进去，就是需要通过长度和模数的关系来对应好不同的循环，然后以此类推？解完题看题解我才发现，这其实是官方题解中环形数组的思路，但是为了实现这个思路需要太多的辅助变量和循环，让我有点头大，就没有继续深入。
- 数组本身有反转这个原地操作，而且反转两次可以变回原来的顺序，也许可以试试。整体反转一次之后数组将会变成：`[7,6,5,4,3,2,1]`，可以发现 `7,6,5` 已经挪到了前面，只需要一次反转就可以来到它们正确的位置上，而对于 `4,3,2,1` 也是如此，只需要再来一次反转。这时一切都明了了——先对 `k` 执行以数组长度为模的取模运算，得到的值减一（因为数组下标从 0 开始）就是数组反转后需要的截断处的元素下标。

以下是最终的题解，使用 Rust 编写：

```rust
impl Solution {
    pub fn rotate(nums: &mut Vec<i32>, k: i32) {
        let len = nums.len();
        let movement = (k as usize) % len;

        nums.reverse();
        nums[0..movement].reverse();
        nums[movement..].reverse();
    }
}
```

## 2. [56. 合并区间](https://leetcode.cn/problems/merge-intervals/)

以数组 `intervals` 表示若干个区间的集合，其中单个区间为 `intervals[i] = [starti, endi]` 。请你合并所有重叠的区间，并返回 *一个不重叠的区间数组，该数组需恰好覆盖输入中的所有区间* 。

**示例 1：**

```
输入：intervals = [[1,3],[2,6],[8,10],[15,18]]
输出：[[1,6],[8,10],[15,18]]
解释：区间 [1,3] 和 [2,6] 重叠, 将它们合并为 [1,6].
```

**示例 2：**

```
输入：intervals = [[1,4],[4,5]]
输出：[[1,5]]
解释：区间 [1,4] 和 [4,5] 可被视为重叠区间。
```

这题理解起来并不复杂，从一开始就有很明显的双指针提示，从第一个区间开始不断扫描前一个区间的右端点和后一个区间的左端点，判断大小关系，然后根据大小关系决定是扩容前一个区间、还是直接 push 后一个区间。

但是为了方便连续的扫描需要加一个排序，耗费 $O(nlogn)$ 时间复杂度，左思右想没想到什么好方法，只能按每个区间的左端点先排序了：

```rust
impl Solution {
    pub fn merge(mut intervals: Vec<Vec<i32>>) -> Vec<Vec<i32>> {
        // 剪枝：对于数组为空的情况
        if intervals.is_empty() {
            return Vec::new();
        }
        // 排序，这里类似 python3 的 lamda，将数组中每个数组的第一个元素当作排序依据
        // 默认是正序，即从小到大
        intervals.sort_by_key(|x| x[0]);
        // 初始化答案的数组，之所以用第一个区间而不是空数组作为初始化，
        // 是因为后续要后续扫描过程中需要有第一个能用来对比的右端点
        let mut res = vec![intervals[0].clone()];
        // 对于原本这个数组的数组，切片还是数组
        for v in &intervals[1..] {
            // 这里用 .last_mut() 可以取出 res 中最后一个元素，并且是可变的
            // 注意这里 res 也是数组的数组，所以取出的元素是数组而非直接的右端点
            // .unwrap() 确保取出的元素非空，挺方便的，不用模式匹配取值了
            let last = res.last_mut().unwrap();
            // 后一个区间的左端点 <= 前一个区间的右端点
            // 将后一个区间的右端点作为前一个区间的左端点
            if v[0] <= last[1] {
                last[1] = last[1].max(v[1]);
            // 否则直接推入这个新区间
            } else {
                res.push(v.clone());
            }
        }

        res
    }
}
```
