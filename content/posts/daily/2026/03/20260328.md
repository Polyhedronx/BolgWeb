## 1. 如何正确处理二分查找的边界值？

在做 [240. 搜索二维矩阵 II](https://leetcode.cn/problems/search-a-2d-matrix-ii/) 的时候，开始是采用了类似二叉树搜索的方法（当然也是最优解），后来看题解还有一个二分查找的方法，原理非常非常简单，就想着也写一遍，结果居然一直在边界条件出错，值得总结一下：

- 循环条件尽量左开右闭，这样就不用讨论右边界为 0 的时候继续循环时、可能出现的 `usize` 无符号类型小于零时溢出的情况了。
- 在左开右闭的情况下，右边界可以不用为了照顾数组最大下标比长度少 1 这一特性，可以直接用数组长度了，降低了心智负担。
- 尽量利用整数除法向下取整的特性，**取左中点而非右中点**，注意 `left + (right - left) / 2` 等价于 `[2left + (right - left)] / 2` 即 `(left + right) / 2`，这就是左中点，使用前者更能处理较大的数据量而不溢出。

最后的代码如下：

```rust
impl Solution {
    pub fn search_matrix(matrix: Vec<Vec<i32>>, target: i32) -> bool {
        // 剪枝1：矩阵为空
        if matrix.is_empty() || matrix[0].is_empty() {
            return false;
        }

        let cols = matrix[0].len();
        for row in matrix {
            // 剪枝2：target超出矩阵边界
            if row[0] > target { break; }
            // 如果 target 大于该行的最大值（即右端点），则跳过该行
            if row[cols - 1] < target { continue; }

            // 初始化二分查找的左右边界指针
            let mut left = 0;
            let mut right = cols;
            while left < right {
                let mid = left + (right - left) / 2;
                // Rust 使用 match 比 if-else 好
                match row[mid].cmp(&target) {
                    std::cmp::Ordering::Equal => return true,
                    // 这里赋值 mid + 1 是为了抛弃左边包括 mid 在内的所有数
                    std::cmp::Ordering::Less => left = mid + 1,
                    // 这里因为 while 条件中是左闭右开
                    // 不需要变成 mid - 1 就已经算抛弃了 mid 及右边所有数
                    std::cmp::Ordering::Greater => right = mid,
                }
            }
        }

        false
    }
}
```
