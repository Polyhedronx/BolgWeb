## 1. 对递归的一些思考

虽然都说在工程实践中使用递归不好，会导致爆栈、数据溢出、维护性变差等等问题，但作为一种解题的通用思路，递归本身也是存在锻炼思维的价值的。我总结了一些关于如何更好思考使用递归解题的要点：

1. **不要在意细节**：也就是说——不要试图用大脑从上而下地模拟，人脑天生对于栈的展开逻辑不是很明确，OI WIKI 有句话说得挺有道理的：“人脑它能模拟几个栈？”。当我们关注抽象的问题本身时，只要抽象出 `n` 和 `n+1` 之间的关系就好。
2. **重点关注特判**：类似数学归纳法，总是在问题最开始的时候处理起来非常简单，而后就越来越复杂——对于一个合理的递归函数，最开始最简单的部分才是真正的基础，基本完成了整个函数核心的输出任务。比如求二叉树的深度，就是在最底层空指针的基础上一层层递归调用的过程中 `+1` 得来的结果。
3. **先找规律**：比如斐波那契数列的问题，当找到 $f(n)$ 和 $f(n+1)$ 之间的关系的时候，结合前面说的特判，问题就自然而然地解决了。如果觉得关系难找，可以先在一个很小但不至于是特判的子问题中模拟运算一下、画画图制制表。

这里给出一道经典的汉诺塔问题的递归解答，如果使用迭代算法，非十分繁琐的代码不能解决：

```c++
#include <iostream>
using namespace std;
// 函数作用：把 n 个盘子从 src 借助 tmp 移动到 dst
// n：盘子总数；src：源柱子；tmp：辅助柱子；dst：目标柱子
void hanoi(int n, char src, char tmp, char dst) {
    // 递归终止条件：只有 1 个盘子时，直接从源移到目标
    if (n == 1) {
        cout << "移动盘子 1 从 " << src << " 到 " << dst << endl;
        return;
    }
    // 1.先把上面 n-1 个盘子 从 src 借助 dst 移到 tmp
    hanoi(n - 1, src, dst, tmp);
    // 2.把最底下第 n 个大盘子 直接从 src 移到 dst
    cout << "移动盘子 " << n << " 从 " << src << " 到 " << dst << endl;
    // 3.把刚才移到 tmp 的 n-1 个盘子 借助 src 移到 dst
    hanoi(n - 1, tmp, src, dst);
}

int main() {
    // 3 个盘子，柱子分别叫 A、B、C
    int n = 3;
    cout << "汉诺塔移动 " << n << " 个盘子的步骤：" << endl;
    // 把 n 个盘子从 A 借助 B 移到 C
    hanoi(n, 'A', 'B', 'C');
    return 0;
}
```
