## 1. vector 下标递归调用防止创建时溢出

今天写 [108. 将有序数组转换为二叉搜索树](https://leetcode.cn/problems/convert-sorted-array-to-binary-search-tree/) 的时候，本来递归写的非常顺手，但是运行的时候报错了 vector 下标越界，检查代码发现只有可能是当 `mid = 0` 的时候，出现了 `nums.begin() = nums.begin() + mid` 的情况，而 cpp 在创建迭代器的时候，本质是创建了一个左闭右开的区间，这就导致了——开头在后面，结尾在前面！这显然是一个非法行为。

```c++
class Solution {
public:
    TreeNode* sortedArrayToBST(vector<int>& nums) {
        // 剪枝：数组为空
        if (nums.empty()) return nullptr;
        // 获取数组中点
        int mid = nums.size() / 2;
        // 堆上创建一个 root 指针，防止函数递归调用的时候出现野指针
        TreeNode* root = new TreeNode(nums[mid]);
        // 切割原数组，这里使用了 cpp 提供的迭代器构造法
        vector<int> leftNum(nums.begin(), nums.begin() + mid);
        vector<int> rightNum(nums.begin() + mid + 1, nums.end());
        // 递归调用
        root->left = sortedArrayToBST(leftNum);
        root->right = sortedArrayToBST(rightNum);
        return root;
    }
};
```

要想改变这个情况，我最开始想的是增加两个 `if-else`，发现不好处理递归调用时的特殊情况，于是转变思路，采取类似二分查找时搜索数组下标的方法，在这里构造了一个 `build()` 函数，通过递归下标来达到安全得到数组值并将其赋给对应节点的结果：

```c++
class Solution {
public:
    TreeNode* sortedArrayToBST(vector<int>& nums) {
        return build(nums, 0, nums.size() - 1);
    }

    TreeNode* build(vector<int>& nums, int left, int right) {
        if (left > right) return nullptr;
        // 这里运用的是在二分查找中学到的左中点防溢出写法
        int mid = left + (right - left) / 2;
        TreeNode* root = new TreeNode(nums[mid]);
        // 递归调用
        root->left = build(nums, left, mid - 1);
        root->right = build(nums, mid + 1, right);
        return root;
    }
};
```

而且最重要的是：**这种方法不拷贝数组，只使用索引，效率更优**。

## 2. 从 [98. 验证二叉搜索树](https://leetcode.cn/problems/validate-binary-search-tree/) 总结 BFS 规律

BFS 全称 Breadth First Search，广度优先搜索，在二叉树这一块的应用颇多，于是总结一下逻辑：

1. 先初始化队列，将需要进行后续核心操作（例如这道题就是判断值和上下界的关系）的元素传入队列中（这里使用了元组传入了节点指针和两个代表界限的 `long long int`。
2. 开始 `while` 循环判断，当队列不为空就一直重复。
3. 在循环内部，取出队首待处理元素 `.front()`，然后将队首元素出列 `.pop()`。
4. 在循环内部，进行核心操作——本题就是判断 + 返回值。
5. 在循环内部，将该节点的左右节点（如果需要按顺序就要注意一下顺序，但本题不用）入队。注意入队最好使用 `.emplace()` 方法，该方法在传入元组的时候会自动优化传入方式，不需要手写 `.push(make_tuple())` 了，同时，对于基础的指针等数据的传入也和 `.push()` 速度相当。

```c++
class Solution {
public:
    bool isValidBST(TreeNode* root) {
        // 使用 bfs 逐层判断
        queue<tuple<TreeNode*, long long, long long>> q;
        q.emplace(root, LLONG_MIN, LLONG_MAX);
        // 当队列不为空，开始处理
        while (!q.empty()) {
            // 取出待处理元素，出队
            auto [node, lower, upper] = q.front();
            q.pop();
            // 判断是否合法
            if (!node) continue;
            if (node->val <= lower || node->val >= upper) return false;
            // 后续元素入队
            q.emplace(node->left, lower, node->val);
            q.emplace(node->right, node->val, upper);
        }
        // 所有元素都通过判断
        return true;
    }
};
```

## 3. 总结一些手动维护栈的 DFS 规律

对于所有需要手动在堆上维护一个栈的 DFS 题目来说，有一些一定不会变的核心操作：

- 外层循环永远不变 `while (curr || !stack.empty())`，只要还有节点没被遍历就继续。
- 由于栈是后进先出，所以想让谁先被遍历到，就后压入栈。
- 所谓前序中序后序，只是为了方便处理问题而套路化的一些方法，更准确的说，其实只和核心处理的代码的位置有关系。比如前序就发生在元素入栈时，中序发生在元素出栈时，后续需要二次处理（比如打标记或者用第二个栈）。

对于中序遍历，我还是使用上面的 [98. 验证二叉搜索树](https://leetcode.cn/problems/validate-binary-search-tree/) 作为例题。总结出来的规律是：

1. 一路向左，沿途所有节点都压入栈，暂不处理。
2. 左无可左时，将顶部元素出栈。
3. 处理出栈的元素。
4. 转向右子树。

第四步这里的逻辑可能不好理解，我最开始也有这种疑惑：明明这个节点可能并没有右节点，为什么可以直接转向？但只要在脑海里构想一些很简单的例子就恍然大悟了，比如 `[2, 1, 3]` 这颗树，向左探到 `1`，右边并无节点，于是开始下一轮循环，此时 `curr` 变成了 `2`，一下就有右节点了。

```c++
class Solution {
public:
    bool isValidBST(TreeNode* root) {
        // 用手动维护栈的中序遍历写法再写一次
        stack<TreeNode*> st;
        TreeNode* curr = root;
        long long lower = LLONG_MIN;
        // 初始化时：元素非空 / 后续操作时：栈非空
        while (curr || !st.empty()) {
            // 一路向左下方下探，并将沿途所有非空节点压入栈
            while (curr) {
                st.push(curr);
                curr = curr->left;
            }
            // 取出栈的顶部元素，等待进行核心比较
            curr = st.top();
            st.pop();
            // 比较，顺便将左区间的边界值改为此节点的值
            if (curr->val <= lower) return false;
            lower = curr->val;
            // 转向右子树
            curr = curr->right;
        }
        return true;
    }
};
```

下面简单写写前序遍历：

```c++
vector<int> preorderTraversal(TreeNode* root) {
    vector<int> res;
    stack<TreeNode*> st;
    TreeNode* curr = root;
    // 标准外层循环不变
    while (curr || !st.empty()) {
        // 一路向左，入栈时处理
        while (curr) {
            // 入栈时处理
            res.push_back(curr->val);
            st.emplace(curr);
            curr = curr->left;
        }
        // 左无可左，出栈，转向右
        curr = st.top();
        st.pop();
        curr = curr->right;
    }
    return res;
}
```

还有使用标记法的后序遍历：

```c++
vector<int> postorderTraversal(TreeNode* root) {
    vector<int> res;
    // 布尔值代表节点指针是否已遇到过
    stack<tuple<TreeNode*, bool>> st;
    TreeNode* curr = root;
    // 标准外层循环不变
    while (curr || !st.empty()) {
        // 一路向左，压入未标记节点
        while (curr) {
            st.emplace(curr, false);
            curr = curr->left;
        }
        // 弹出栈顶
        auto [node, visited] = st.top();
        st.pop();

        if (!visited) {
            // 第一次访问：重新压入已标记节点，先压右、再压左
            st.emplace(node, true);
            // 转向右子树
            curr = node->right;
        } else {
            // 第二次访问：此时才处理
            res.push_back(node->val);
        }
    }
    return res;
}
```
