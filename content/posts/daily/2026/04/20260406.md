## 1. 使用宏来帮助生成逻辑复杂的表达式

有时候来多层循环中可能出现一些逻辑复杂的表达式，比如 Tsoding 直播中自定义的矩阵类型的打印函数：

```c
#include <stdio.h>
#include <stdlib.h>

typedef struct {
    size_t rows;
    size_t cols;
    float *es;
} Mat;

// 宏的本质是纯文本替换，没有额外函数开销，但是因为没有类型检查所以安全型差一点
// 这里的参数括号是为了防止参数是表达式而打乱运算顺序
// 这里的整体括号是为了防止多层宏调用而打乱运算顺序
#define MAT_AT(mat, i, j)  ((mat).es[(i) * (mat).cols + (j)])

void mat_print(Mat m) {
    for (size_t i = 0; i < m.rows; ++i) {
        for (size_t j = 0; j < m.cols; ++j) {
            printf("%8.2f ", MAT_AT(m, i, j));
        }
        printf("\n");
    }
}
```

> 注意：由于是纯文本替换，所以如果宏参数是自增 / 自减，会在某些循环内部被执行多次，所以尽量避免在宏内部使用这些。

## 2. 在 C++ 或 Rust 中有什么类似宏功能的实现？

由于纯文本替换既不安全也不优雅，我就理所当然的去查了查在更现代的语言中，有没有类似的实现——答案不出我所料：有的。

### C++ 中的运算符重载

直接让矩阵支持 `m(i,j)` 语法，比宏 `MAT_AT(m,i,j)` 更简洁，**零开销、类型安全、无副作用**。

```c++
#include <cstdio>
#include <cstddef>

struct Mat {
    size_t rows;
    size_t cols;
    float* es;
    // 编译期内联函数，重载()运算符
    // 等价于MAT_AT宏，但完全安全，零运行时开销
    constexpr float& operator()(size_t i, size_t j) noexcept {
        return es[i * cols + j];
    }
    // 只读访问，常量对象版本
    constexpr const float& operator()(size_t i, size_t j) const noexcept {
        return es[i * cols + j];
    }
};

// 打印函数：直接用 m(i,j)，比宏更自然
void mat_print(const Mat& m) {
    for (size_t i = 0; i < m.rows; ++i) {
        for (size_t j = 0; j < m.cols; ++j) {
            printf("%8.2f ", m(i, j));
        }
        printf("\n");
    }
}
```

这里有几点相对于 C 中的宏的额外优势：

1. **类型安全**：如果运算符内部传入的参数不是 `size_t` 类型，就会直接报错。
2. **没有副作用**：对于 `m(i++, j)` 这样的自增 / 自减参数，只会运算一次。
3. **运行时无成本**：让表达式在编译期就算出结果，在运行时零开销。
4. **语法更自然**：`m(i, j)` 表示矩阵对应行列的元素要比 `MAT_AT(m, i, j)` 自然。

### Rust 中可通过实现 Trait 来做到某些语法特征

```rust
#[derive(Debug)]
struct Mat {
    rows: usize,
    cols: usize,
    es: Vec<f32>,
}

// 实现标准库 Index 特征，支持 m[(i,j)] 索引
// 完全替代C宏，安全、零开销、内存安全
impl std::ops::Index<(usize, usize)> for Mat {
    type Output = f32;

    fn index(&self, (i, j): (usize, usize)) -> &Self::Output {
        &self.es[i * self.cols + j]
    }
}

// 打印函数：直接用 m[(i,j)]，原生语法
fn mat_print(m: &Mat) {
    for i in 0..m.rows {
        for j in 0..m.cols {
            print!("{:8.2} ", m[(i, j)]);
        }
        println!();
    }
}
```
