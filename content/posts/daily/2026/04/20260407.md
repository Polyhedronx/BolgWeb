## 1. 二叉树的右视图：层序遍历思想和优化

我最开始的想法就是：这样一层一层看最右边的元素值，天然就很符合层序遍历的思想，但在实际操作中会发现有许多障碍——比如如何保证返回的是最右边的元素？在单次 `while` 循环中有点难以想到，于是想着将单层的所有元素值全部储存到一个临时的数组中，然后在循环的末尾使用 `temp.size() - 1` 对应的索引取出最右边的元素值。下面是代码实现：

```c++
class Solution {
public:
    vector<int> rightSideView(TreeNode* root) {
        // 感觉挺适合层序遍历的，每一层返回最后一个就好了
        vector<int> res;
        if (!root) return res;
        queue<TreeNode*> q;
        q.emplace(root);
        while (!q.empty()) {
            vector<int> temp;
            int size = q.size();
            for (int i = 0; i < size; ++i) {
                TreeNode* curr = q.front();
                q.pop();
                if (curr->left) q.emplace(curr->left);
                if (curr->right) q.emplace(curr->right);
                temp.push_back(curr->val);
            }
            res.push_back(temp[temp.size() - 1]);
        }
        return res;
    }
};
```

但后来发现，实际上并不需要去开辟额外的数组空间，既然已经有了靠 “索引” 去取元素值的思想，就可以在每次 `while` 循环的过程中去判断现在是否已经到了对应的索引处：

```c++
class Solution {
public:
    vector<int> rightSideView(TreeNode* root) {
        vector<int> res;
        if (!root) return res;
        queue<TreeNode*> q;
        q.emplace(root);
        
        while (!q.empty()) {
            int size = q.size();
            for (int i = 0; i < size; ++i) {
                TreeNode* curr = q.front();
                q.pop();
                
                // 直接判断是否是当前层最后一个节点
                if (i == size - 1) {
                    res.push_back(curr->val);
                }
                
                if (curr->left) q.emplace(curr->left);
                if (curr->right) q.emplace(curr->right);
            }
        }
        return res;
    }
};
```
