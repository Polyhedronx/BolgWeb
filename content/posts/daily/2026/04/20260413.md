## 1. 如何得到一个二进制数的位数？

### 心路历程

在做 CS：APP 的 Lab 的时候发现这个函数很难想。

我最开始的想法是：如何将正数和负数统一进一个可比较的体系？毕竟负数的符号位是永远为 1 的，通过向右移位再向左移位也不好——因为负数的计数位的规则与正数不一样，比如四 bit 的数 `1000` 可不是 `0000` 的相反数，前者在十进制表示 `-8`，而后者是 `0`，二进制里 `0111` 表示十进制的 `7` 才更接近。这里用这一个小例子就可以看出：**应该对负数按位取反**。

但如何只对负数按位取反而不对正数这么做呢？让我回忆起了 “掩码” 这个东西，如果让正数的掩码全为 `0`，负数的掩码全为 `1` 就能使用简单的**异或操作**了。很巧的是：用惯常的符号位移位操作就可以制造出这样的掩码，只需要 `x >> 31` 就好了。

接下来是另一个核心：**如何比较？**

还是从最简单的例子出发，比如 `00000000 00000000 11111111 11111111` 这样一个 32 位正数，使用二分法试试看：

- 向右移动 16 位变成 `00000000 00000000 00000000 00000000` 显然为 `0`，不用加位数。
- 向右移动 8 位变成 `00000000 00000000 00000000 11111111`，显然有数，要加 8 位。这里可以对移位的数据做**两次逻辑非**，将所有不为零的归化为 `1`，为零的规划为 `0`。
- 因为至少有 8 位，所以下一次二分时需要把这个 8 位也算在内，不然无法扩大比较范围——于是向右移动 8 + 4 = 12 位变成 `00000000 00000000 00000000 00001111`，显然也有数，位数要再加 4 位变成 12 位。
- 这里向右移动 12 + 2 = 14 位变成 `00000000 00000000 00000000 00000011`，显然有数，操作不再赘述。
- 这里向右移动 14 + 1 = 15 位变成 `00000000 00000000 00000000 00000001`，显然有数，操作不再赘述。
- 因为最高位在第 `bits` 位（从 0 开始），所以有效位是 `bits + 1` 位。再加上一位符号位，就是 `bits + 2` 位。

但还有**特殊情况**：比如给定的参数是 `0` 或 `-1`，这时补码就只需要 1 位，而此时 `pos = 0; !pos = 1`，用这个作为最后返回值的偏置就可以了。

```c
/* howManyBits - return the minimum number of bits required to represent x in
 *             two's complement
 *  Examples: howManyBits(12) = 5
 *            howManyBits(298) = 10
 *            howManyBits(-5) = 4
 *            howManyBits(0)  = 1
 *            howManyBits(-1) = 1
 *            howManyBits(0x80000000) = 32
 *  Legal ops: ! ~ & ^ | + << >>
 *  Max ops: 90
 *  Rating: 4
 */
int howManyBits(int x) {
  // 负数取反，正数不变
  int pos = x ^ (x >> 31);
  int bits = 0;
  // 二分法取位数
  bits += (!!(pos >> 16) << 4);
  bits += (!!(pos >> (bits + 8)) << 3);
  bits += (!!(pos >> (bits + 4)) << 2);
  bits += (!!(pos >> (bits + 2)) << 1);
  bits += !!(pos >> (bits + 1));
  // 处理特殊情况：pos 为 0 或者 -1
  return bits + 2 - (!pos);
}
```

### 总结

1. **注重掩码 `mask` 的特性**：取反、单独显示等等。
2. **二分法在位操作中定位**：面对 32 位这样的长序列，不必逐位检查，通过从高到低（16 位 -> 8 位 -> 4 位 -> 2 位 -> 1 位）的二分试探，能快速锁定最高有效位的位置，将操作数控制在常数级。
3. **善用 `!!` 做归一化**：双重逻辑非能将任意非零值转换为 `1`、零保持 `0`，这让位操作的结果可以直接用于加法计数，避免了复杂的条件判断。
4. **特殊情况优先用数学偏置处理**：对于 `0` 和 `-1` 这类边界值，不必写 `if-else` 分支，通过观察 `!pos` 的值（特殊情况为 `1`，普通情况为 `0`），将其作为返回值的修正项，代码更简洁也更符合位操作的约束。
